## 2. 브랜치 네이밍 규칙

### 형식
```
feat/[태스크번호]-[설명]-[MMDD]
```

### 예시

| 태스크 | 브랜치명 |
|---|---|
| GitHub 저장소 생성 | `feat/001-github-setup-0618` |
| Vite 프로젝트 초기화 | `feat/002-vite-init-0618` |
| Supabase 스키마 적용 | `feat/003-supabase-schema-0618` |
| PDF 파서 연동 | `feat/004-pdf-parser-0619` |
| 문제 풀기 UI | `feat/005-question-ui-0619` |
| AI 해설 연동 | `feat/006-ai-explanation-0619` |
| 오답노트 저장 | `feat/007-wrong-note-0620` |
| 공유 카드 기능 | `feat/008-share-card-0620` |
| Vercel 배포 | `feat/009-deploy-0620` |

### 접두어 규칙
| 접두어 | 언제 사용 |
|---|---|
| `feat/` | 새 기능 개발 |
| `fix/` | 버그 수정 |
| `hotfix/` | 배포 후 긴급 수정 |
| `chore/` | 설정, 패키지 등 비기능 작업 |
| `docs/` | 문서 수정만 할 때 |

## 3. 작업 플로우

### 태스크 시작할 때
```bash
# develop 브랜치 최신화
git checkout develop
git pull origin develop

# 태스크 브랜치 생성
git checkout -b feat/004-pdf-parser-0619
```

### 작업 중 커밋
```bash
git add .
git commit -m "feat: pdf.js 파서 기본 구현"
git commit -m "feat: Claude API 문제 재생성 연동"
git commit -m "fix: pdf workerSrc 오류 수정"
```

### 태스크 완료 후 PR
```bash
# develop으로 push
git push origin feat/004-pdf-parser-0619

# GitHub에서 PR 생성
# Base: develop ← Compare: feat/004-pdf-parser-0619
# PR 제목: [feat] PDF 파서 연동 (#004)
```

### develop → main 머지 (배포 시)
```bash
# 모든 태스크 완료 후
git checkout main
git merge develop
git push origin main
# → Vercel 자동 배포 시작
```

## 4. 커밋 메시지 컨벤션

### 형식
```
[타입]: [무엇을 했는지 한국어로]
```

### 타입 목록
| 타입 | 의미 | 예시 |
|---|---|---|
| `feat` | 새 기능 | `feat: PDF 업로드 파싱 기능 추가` |
| `fix` | 버그 수정 | `fix: 오답노트 저장 안 되는 문제 수정` |
| `style` | UI/스타일 변경 | `style: 다크 테마 색상 토큰 적용` |
| `refactor` | 코드 구조 개선 | `refactor: useClaudeAI 훅으로 분리` |
| `chore` | 설정/패키지 | `chore: tailwind 설정 추가` |
| `docs` | 문서 수정 | `docs: 사용자가이드 FAQ 추가` |
| `hotfix` | 긴급 수정 | `hotfix: 배포 후 API 키 오류 수정` |

## 5. 버전 관리 (Semantic Versioning)

### 형식
```
v[Major].[Minor].[Patch]
```

| 버전 타입 | 언제 올리나 | 예시 |
|---|---|---|
| `Patch` (+0.0.1) | 버그 수정 | v1.0.0 → v1.0.1 |
| `Minor` (+0.1.0) | 기능 추가 | v1.0.0 → v1.1.0 |
| `Major` (+1.0.0) | 대규모 변경 | v1.0.0 → v2.0.0 |

### CertPass AI 버전 로드맵
| 버전 | 내용 | 목표일 |
|---|---|---|
| `v0.1.0` | MVP — 문제풀기 + AI 해설 | 6/20 |
| `v0.2.0` | 오답노트 + 공유 카드 | 6/20 |
| `v1.0.0` | 네이트 앱 배포 완료 | 6/20 |
| `v1.1.0` | 소셜 로그인 + 통계 차트 | 추후 |
| `v2.0.0` | 타 자격증 확장 | 추후 |

### 태그 만들기 (배포 시)
```bash
git tag -a v1.0.0 -m "feat: CertPass AI MVP 배포"
git push origin v1.0.0
```
